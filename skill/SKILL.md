# Elyonize

## Overview

This skill has two modes:

**Elyonize (Default Decode Mode)**: Produces a clean, analytical breakdown with component meanings, overall meaning, and short interpretation.

**Elyon Analyze (Detailed Mode)**: Produces the full technical analysis with Sound Mapping Tables, root notes, and Lexical Entry.

## When to Activate (Easter Egg Mode)

**1. Elyonize (Default Decode Mode)**
- Triggered by: "Elyonize", "Elyonize this", "Elyonize [word/phrase]", or similar.
- Produces a clean, analytical breakdown similar to linguistic decoding:
  - Word/phrase component breakdown
  - Overall meaning
  - Short interpretive/contextual note
- This is the **default** behavior for "Elyonize".

**2. Elyon Analyze (Detailed Mode)**
- Triggered by: "Elyon Analyze", "Elyonize Analyze", "detailed Elyonize", or "full breakdown".
- Produces the full technical analysis:
  - Sound Mapping Tables (with IPA + X-SAMPA)
  - Root & Cognate Notes (including Hoffmann transcription for Avestan)
  - Lexical Entry with etymology flavor + cross-linguistic notes
  - Plain Language Restoration

The deeper linguistic mechanics are only shown in **Elyon Analyze** mode unless the user specifically asks.

## Elyonize Output Template (Default - Light Decode Mode)

Use this clean format:

**Breakdown:**
- [Component] = meaning / function
- [Component] = meaning / function

**Overall Meaning:**
"[Clear, natural rendering of the full phrase]"

**Interpretation:**
Short paragraph explaining the conceptual or metaphorical meaning in context.

## Elyon Analyze Output Template (Detailed Technical Mode)

Use the full format with Sound Mapping Tables, Root & Cognate Notes, Lexical Entry, and Plain Language Restoration.

## Core Process (Follow Every Time)

1. **Analyze the Input**
   - Break the source word or phrase into syllables and key sounds.
   - Note stress patterns, length, tone (playful, majestic, mysterious, technical, etc.), and any context provided.

2. **Apply Resonant Sound Mapping**
   - Use established phonetic correspondence patterns (consult internal reference when needed for consistency on vowels, consonants, and clusters).
   - Adapt sounds for euphony and natural flow in the target phonetic system while keeping the result recognizable and pleasing.
   - Favor combinations that sound harmonious and elevated.

3. **Construct the Transformed Form(s)**
   - Generate 1–3 options:
     - **Primary (Close Resonance)**: Strongest phonetic match with minimal added invention.
     - **Enhanced (Layered / Compound)**: Builds on the primary with elegant additions, suffixes, or compounding for greater depth and natural rhythm.
     - **Interpretive / Evocative** (optional): A version that leans into meaning or atmosphere while staying true to the sound.
   - Ensure the result feels cohesive and intentional.

4. **Choose Output Mode**
   - Default ("Elyonize"): Use the **light Decode template**.
   - "Elyon Analyze" or explicit detail request: Use the **full technical template**.

5. **Add Context and Polish**
   - Include concise notes on why certain mappings or constructions were chosen.
   - Note any pleasing resonances or elevated qualities.
   - If transforming multiple items or a longer phrase, present them clearly as a set or suggest a title-style rendering.

6. **Format Consistently**
   - Follow the correct template based on the mode triggered.
   - Use clean headings and structure for readability.
   - Offer refinements at the end ("Want additional variations, a different tone, or a full phrase rendering?").
